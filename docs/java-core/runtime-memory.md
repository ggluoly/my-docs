# 运行时内存结构

> 本页默认以 JDK 8 为技术基线。

JVM 内存结构是分析 OOM、栈溢出、GC 和类加载问题的基础。JDK 8 的关键变化是移除永久代，使用元空间存放类元信息。

## 内存区域

```text
JVM 内存
├── 堆（Heap）
│   ├── 新生代（Young）
│   │   ├── Eden
│   │   ├── Survivor From
│   │   └── Survivor To
│   └── 老年代（Old）
├── 方法区 / 元空间（Metaspace）
├── 虚拟机栈（Java Virtual Machine Stack）
├── 本地方法栈（Native Method Stack）
└── 程序计数器（Program Counter Register）
```

## 堆

堆是线程共享区域，主要存放对象实例和数组，是 GC 的主要工作区域。

JDK 8 常见分代：

- 新生代：大多数新对象优先分配在 Eden。
- Survivor：Minor GC 后仍存活的对象会在两个 Survivor 区之间复制。
- 老年代：长期存活对象、大对象或晋升对象进入老年代。

常见问题：

- `OutOfMemoryError: Java heap space`
- Full GC 频繁
- 大对象分配导致老年代压力增大

## 虚拟机栈

虚拟机栈是线程私有区域。每个方法调用都会创建一个栈帧，栈帧中包含局部变量表、操作数栈、动态链接和方法返回地址等信息。

常见问题：

- 递归过深导致 `StackOverflowError`。
- 线程数过多导致本地内存压力增大。

## 本地方法栈

本地方法栈服务于 native 方法。普通业务开发较少直接接触，但使用 JNI、压缩库、加密库、图像处理库时可能间接涉及。

## 程序计数器

程序计数器是线程私有区域，记录当前线程正在执行的字节码指令位置。

特点：

- 占用内存很小。
- 线程切换后依靠它恢复执行位置。
- 是 JVM 规范中唯一没有规定 `OutOfMemoryError` 的区域。

## 方法区与元空间

方法区是 JVM 规范概念，用于存放类元信息、运行时常量池、方法信息等。

JDK 8 中 HotSpot 移除了永久代，改用元空间：

- 元空间使用本地内存。
- 默认情况下会根据系统内存动态增长。
- 类加载过多、动态代理过多、类加载器泄漏都可能导致 `Metaspace OOM`。

常见参数：

```text
-XX:MetaspaceSize=256m
-XX:MaxMetaspaceSize=512m
```

## 对象分配简化流程

典型对象分配流程：

```text
new 对象
-> 优先分配到 Eden
-> Eden 空间不足触发 Minor GC
-> 存活对象进入 Survivor
-> 多次存活或空间不足晋升老年代
-> 老年代空间不足可能触发 Full GC
```

JIT 逃逸分析可能做标量替换、锁消除等优化，因此不是所有对象都一定产生实际堆分配，但从业务排查角度仍应优先按堆对象理解。

## 实践建议

- 生产环境显式设置堆大小，避免默认值不符合容器资源限制。
- `-Xms` 和 `-Xmx` 建议设置为相同值，减少运行期动态扩容。
- 关注堆、元空间、线程数和直接内存指标。
- OOM 时保留堆转储，否则很难还原现场。
