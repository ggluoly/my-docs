# Java 18-21 特性

> Java 基础原理和存量项目维护以 JDK 8 为讲解基线；在框架、依赖、Agent 和部署环境兼容时，Java 21 是现代 Spring Boot 3.x 新项目优先评估的 LTS 版本。

Java 18 到 21 的重点是虚拟线程、模式匹配完善、集合顺序抽象和低延迟 GC。Java 21 是 LTS 版本，也是当前新项目常见升级目标。

## 特性总览

| 版本 | 重点特性 | 说明 |
| --- | --- | --- |
| Java 18 | UTF-8 成为默认字符集 | 跨平台文本处理更稳定 |
| Java 18 | Simple Web Server | JDK 自带简单静态文件服务器 |
| Java 18 / 21 | Pattern Matching for switch | Java 21 正式可用 |
| Java 19 / 21 | Record Patterns | Java 21 正式可用 |
| Java 19 / 21 | 虚拟线程 | Java 21 正式可用 |
| Java 21 | Sequenced Collections | 统一集合首尾访问能力 |
| Java 21 | Generational ZGC | ZGC 支持分代 |
| Java 21 | Scoped Values 预览 | 安全传递上下文的候选方案 |
| Java 21 | Structured Concurrency 预览 | 结构化管理并发任务 |
| Java 21 | String Templates 预览 | 后续未作为正式特性延续，不建议依赖 |

## UTF-8 默认字符集

Java 18 将 UTF-8 作为默认字符集，减少不同操作系统默认编码不一致导致的问题。

JDK 8 项目仍要显式指定编码，例如文件读写、HTTP 响应、数据库连接和 Maven 编译参数。

## 虚拟线程

虚拟线程是 Java 21 正式特性，由 JVM 调度，创建和切换成本远低于平台线程。

适合：

- HTTP 调用。
- RPC 调用。
- 数据库访问。
- 文件和网络 IO。
- 大量阻塞式 IO 任务。

不适合：

- CPU 密集型计算提速。
- 绕过下游容量限制。
- 替代连接池、限流和超时控制。

示例：

```java
try (var executor = Executors.newVirtualThreadPerTaskExecutor()) {
    Future<User> userFuture = executor.submit(() -> userClient.getUser(userId));
    Future<List<Order>> orderFuture = executor.submit(() -> orderClient.getOrders(userId));

    User user = userFuture.get();
    List<Order> orders = orderFuture.get();
}
```

注意：这不是 JDK 8 能力；维护 JDK 8 存量项目时，并发编程仍以平台线程和线程池为基础。

## Pattern Matching for switch

Java 21 正式支持 switch 模式匹配：

```java
String text = switch (obj) {
    case String s -> s;
    case Integer i -> String.valueOf(i);
    case null -> "null";
    default -> "unknown";
};
```

适合处理有限类型分支，配合 sealed class 更清晰。

## Record Patterns

Record Patterns 可以在模式匹配中解构 record：

```java
if (obj instanceof Point(int x, int y)) {
    System.out.println(x + "," + y);
}
```

这进一步增强了 record 的数据建模能力。

## Sequenced Collections

Java 21 新增顺序集合接口：

- `SequencedCollection`
- `SequencedSet`
- `SequencedMap`

统一提供首尾访问、反向视图等能力，解决过去不同集合首尾操作 API 不统一的问题。

## Generational ZGC

Java 21 中 ZGC 支持分代，进一步降低低延迟 GC 的使用门槛。

这属于后续版本 JVM 能力，不影响 JDK 8 存量项目。JDK 8 中重点仍是 Parallel、CMS 和 G1。

## 升级注意事项

- Java 21 是 LTS，可以作为新项目升级目标。
- 虚拟线程上线前要验证监控、链路追踪、连接池、限流和下游容量。
- 不要因为虚拟线程便宜就无限制创建下游请求。
- JDK 8 项目学习虚拟线程时，应把它视为升级方向，而不是当前可用能力。
