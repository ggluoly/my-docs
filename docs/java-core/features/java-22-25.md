# Java 22-25 特性

> Java 基础原理和存量项目维护以 JDK 8 为讲解基线，现代 Spring Boot 3.x 新项目通常选择 Java 17 / 21。本页内容偏前沿，生产采用前必须验证框架、依赖、Agent、CI/CD 和监控链路兼容性。

Java 22 到 25 继续围绕简化入门语法、增强模式匹配、改进原生互操作、提升可观测性和优化运行时启动性能演进。Java 25 是多数发行版关注的新一代 LTS。

## 特性总览

| 版本 | 重点特性 | 说明 |
| --- | --- | --- |
| Java 22 | Foreign Function & Memory API | 正式提供外部函数和堆外内存访问 API |
| Java 22 | Unnamed Variables & Patterns | 用 `_` 表达不关心的变量或模式 |
| Java 22 | 多文件源码直接运行 | `java` 命令可运行多个源码文件组成的小程序 |
| Java 22 / 24 | Class-File API | Java 24 正式可用 |
| Java 22 / 24 | Stream Gatherers | Java 24 正式可用 |
| Java 23 | Markdown 文档注释 | Javadoc 支持 Markdown 风格注释 |
| Java 23 | Generational ZGC 默认启用 | ZGC 分代模式成为默认 |
| Java 24 | 永久禁用 Security Manager | 老项目需要迁移安全边界设计 |
| Java 24 | 虚拟线程 synchronized 友好性改进 | 降低虚拟线程 pinning 风险 |
| Java 24 / 25 | AOT 相关能力 | 改善启动性能和预热体验 |
| Java 25 | Scoped Values | 正式可用，适合替代部分 `ThreadLocal` 场景 |

## Foreign Function & Memory API

FFM API 提供标准方式访问外部函数和堆外内存，可替代部分 JNI 场景。

适合：

- 调用 C 库。
- 高性能堆外内存访问。
- 与系统级能力集成。

普通业务系统不应为了追新而使用它，除非有明确的原生互操作需求。

## Class-File API

Class-File API 为读写 class 文件提供标准 API，有利于字节码工具、框架和分析工具生态。

业务开发者通常不直接使用，但框架、Agent、编译器插件和代码分析工具会受益。

## Stream Gatherers

Stream Gatherers 扩展 Stream 中间操作能力，适合窗口、批处理、自定义聚合等高级流水线场景。

JDK 8 存量项目仍使用现有 Stream API 和 Collector 机制。

## Scoped Values

Scoped Values 在 Java 25 正式可用，适合在线程和虚拟线程中传递只读上下文。

它可以替代部分 `ThreadLocal` 使用场景，尤其是在虚拟线程环境下更容易控制生命周期。

## Module Import Declarations

Java 25 支持模块导入声明：

```java
import module java.base;
```

它可以导入模块导出的包，主要改善教学、小程序和模块化代码的导入体验。

## Compact Source Files and Instance Main Methods

Java 25 降低编写小程序的样板代码，对教学和脚本式小工具更友好。

企业后端主项目仍建议保持清晰的包结构、类结构和工程规范。

## AOT 相关能力

Java 24 到 25 引入 AOT 缓存、命令行易用性和方法 profiling 等方向，目标是改善启动性能和预热体验。

服务端应用可以关注：

- 启动时间。
- 冷启动性能。
- Serverless 或弹性扩缩容场景。
- 镜像构建和部署链路兼容性。

## 预览、孵化和实验特性

Java 22-25 中仍有不少预览、孵化或实验特性，例如：

- 结构化并发。
- 原始类型模式匹配。
- Vector API。
- 部分 JFR 增强。
- 稳定值。

学习时要区分“正式可用”和“仍在演进”。生产项目不要把预览特性作为核心代码基线。

## 升级注意事项

- Java 25 可作为新一代 LTS 关注，但采用前要等待生态成熟。
- 核心框架、数据库驱动、监控 Agent、构建插件都要做兼容性验证。
- JDK 8 项目不要把 Java 22-25 特性写入主干代码，除非已经完成运行时基线升级。
