# Java 12-17 特性

> Java 基础原理和存量项目维护以 JDK 8 为讲解基线；Java 17 是 Spring Boot 3.x 的最低 Java 版本，也是现代新项目的稳健 LTS 选择。

Java 12 到 17 持续增强语言表达能力。Java 17 是 LTS 版本，也是 Spring Boot 3.x 的最低 Java 基线。

## 特性总览

| 版本 | 重点特性 | 说明 |
| --- | --- | --- |
| Java 12 / 14 | switch 表达式 | Java 14 正式可用 |
| Java 13 / 15 | 文本块 | Java 15 正式可用 |
| Java 14 | Helpful NullPointerException | NPE 信息更明确 |
| Java 14 / 16 | instanceof 模式匹配 | Java 16 正式可用 |
| Java 14 / 16 | Record | Java 16 正式可用 |
| Java 15 | ZGC / Shenandoah 转正 | 低延迟 GC 进入生产可用阶段 |
| Java 15 | Nashorn 移除 | 内置 JavaScript 引擎被移除 |
| Java 16 | Stream.toList() | 更简洁收集 Stream 结果 |
| Java 17 | sealed class | 限制继承范围 |
| Java 17 | JDK 17 LTS | 现代 Spring Boot 项目常见基线 |

## switch 表达式

Java 14 中 switch 表达式正式可用：

```java
String type = switch (status) {
    case 1 -> "NEW";
    case 2 -> "PROCESSING";
    case 3 -> "DONE";
    default -> "UNKNOWN";
};
```

相比传统 `switch`，它可以返回值，并减少 `break` 漏写导致的问题。

## 文本块

Java 15 正式引入文本块：

```java
String sql = """
    select id, name
    from user
    where status = 1
    """;
```

适合 SQL、JSON、HTML 等多行字符串。

## Helpful NullPointerException

Java 14 增强 NPE 信息，可以指出具体哪个表达式为 `null`。

这对线上日志排查很有帮助，但不能替代必要的参数校验和空值设计。

## instanceof 模式匹配

Java 16 正式可用：

```java
if (obj instanceof User user) {
    return user.getName();
}
```

减少显式强转，提高类型判断代码可读性。

## Record

Java 16 正式引入 Record：

```java
public record UserView(Long id, String name, Integer age) {
}
```

适合：

- DTO
- VO
- 查询结果
- 配置项
- 不可变数据载体

不适合承载复杂可变领域对象。

## Sealed Class

Java 17 正式引入密封类：

```java
public sealed interface PayCommand permits WechatPayCommand, AliPayCommand {
}
```

它限制允许哪些类继承或实现当前类型，适合表达有限类型层次。

## Stream.toList()

Java 16 增加 `Stream.toList()`：

```java
List<String> names = users.stream()
    .map(User::getName)
    .toList();
```

注意：`Stream.toList()` 返回的列表不可修改。JDK 8 代码仍应使用 `collect(Collectors.toList())`。

## 升级注意事项

- Java 17 是 Spring Boot 3.x 最低基线。
- JDK 内部 API 强封装更严格，老项目升级前要排查非法反射。
- `record`、`sealed`、switch 表达式会明显改变编码风格，团队需要统一规范。
- 如果存在老的 Nashorn 依赖，需要替换 JavaScript 执行方案。
