# 消息与事务

这一层解决微服务之间的**异步通信**和**跨服务数据一致性**两大问题。

消息队列负责异步解耦、削峰填谷、事件驱动；分布式事务负责在多个服务、多个数据库之间保证数据最终一致。

## 本栏目内容

| 文档 | 说明 |
| --- | --- |
| [RocketMQ](./rocketmq) | 业务消息、事务消息、延迟消息，国内企业和电商常用 |
| [Kafka](./kafka) | 高吞吐日志、埋点、流式数据，适合大数据和实时计算 |
| [Apache Pulsar](./pulsar) | 云原生消息与流式数据平台，适合多租户、消息与流统一、分层存储场景 |
| [Seata 分布式事务](./seata) | 跨服务、跨数据库的事务一致性方案 |

RabbitMQ 不新增独立技术页，仅在下表中作为传统 AMQP 选型参考，并保留在[消息队列面试题](/interview/mq)中作为面试考点。

## MQ 对比与选型

| 组件 | 核心定位 | 主要优势 | 典型场景 | 本站定位 |
| --- | --- | --- | --- | --- |
| `RocketMQ` | 业务消息中间件 | 事务消息、延迟消息、顺序消息能力贴近交易业务 | 订单、支付、电商、业务事件与最终一致性 | 正式技术页 |
| `Kafka` | 高吞吐事件流平台 | 吞吐高、日志与流处理生态成熟 | 日志、埋点、CDC、实时计算、数据管道 | 正式技术页 |
| `Pulsar` | 云原生消息与流式数据平台 | 多租户、计算存储分离、分层存储、多订阅模式 | 平台型系统、跨地域、多租户、消息与流统一 | 正式技术页 |
| `RabbitMQ` | 传统 AMQP 消息队列 | 路由模型灵活，适合通知和轻量异步任务 | 中小系统、传统 AMQP 集成 | 选型参考与面试考点，不设独立页 |

`Spring Cloud Stream` 是消息中间件抽象，不是另一种 MQ；是否采用取决于团队是否需要统一 Binder 编程模型。

选型速记：

```
交易和业务消息：RocketMQ
日志和大数据：Kafka
云原生多租户和消息流统一：Pulsar
传统 AMQP 和轻量异步任务：RabbitMQ（参考）
```

```
分布式事务：优先本地事务 → 消息最终一致性 → 复杂场景再用 Seata
```

> 分布式事务不能滥用，能用本地事务或最终一致性解决的，就不要引入 Seata。
