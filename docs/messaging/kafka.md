# Kafka

## 是什么

Kafka 是高吞吐的分布式消息系统，擅长处理日志、埋点、流式数据，适合大数据、日志采集、实时计算场景。

## 解决什么问题

和其他消息队列一样，Kafka 用于异步解耦、削峰填谷、事件驱动等，但它的强项在高吞吐和流式数据处理。

## 定位差异

Kafka 更偏高吞吐日志、事件流和实时计算生态，RocketMQ 更偏交易类业务消息，Pulsar 更偏多租户和计算存储分离的平台型架构。RabbitMQ 仅作为传统 AMQP 方案参考与面试考点。完整对比见 [消息与事务：MQ 对比与选型](/messaging/#mq-对比与选型)。

## 选型建议

当系统涉及大量日志采集、用户行为埋点、实时流式计算时，Kafka 是常见选择。它也常配合实时计算引擎（如 Flink、Kafka Streams）使用，详见[数据分析](../deploy/data-analysis)相关章节。
