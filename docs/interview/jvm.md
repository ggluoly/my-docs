# JVM 高频题

本页整理 JVM 面试高频问题，覆盖 JVM 组成、运行时数据区、类加载、垃圾回收、对象晋升和调优工具。回答时要能画出内存结构，并结合线上排查说明工具和参数。

## JVM 组成

### JVM 主要由哪些部分组成？

#### 标准回答

JVM 主要由类加载子系统、运行时数据区、执行引擎和本地方法接口组成。类加载器负责加载 class；运行时数据区负责存储对象、栈帧、类元信息等；执行引擎负责解释执行或 JIT 编译；本地方法接口用于调用底层 native 方法。

### JVM 内存结构是什么？

#### 标准回答

JVM 运行时内存主要包括堆、虚拟机栈、本地方法栈、程序计数器和方法区 / 元空间。堆是对象分配和 GC 的主要区域；栈保存方法调用栈帧；程序计数器记录当前线程执行位置；元空间存放类元信息。

![JVM 运行时数据区](/images/interview/jvm/jvm-runtime-data-areas.webp)

上图展示了 JVM 运行时数据区的核心组成。面试回答时要能区分线程共享区域和线程私有区域：堆和方法区 / 元空间通常是线程共享的，虚拟机栈、本地方法栈和程序计数器是线程私有的。

### 堆和栈有什么区别？

#### 标准回答

堆是线程共享区域，主要存放对象实例和数组，是 GC 管理的核心区域；虚拟机栈是线程私有区域，存放方法调用栈帧，包括局部变量表、操作数栈、动态链接和返回地址。堆溢出通常是对象太多或泄漏，栈溢出通常是调用层级过深或栈帧过大。

#### 常见追问

### 队列和栈有什么区别？

#### 标准回答

队列是先进先出 FIFO，常用于任务排队、消息缓冲和线程池任务队列；栈是后进先出 LIFO，常用于方法调用、表达式计算和回溯场景。JVM 虚拟机栈体现的就是方法调用的入栈和出栈过程。

## 类加载

### 双亲委派模型是什么？

#### 标准回答

类加载请求会先委派给父加载器，父加载器无法加载时才由子加载器处理。这样可以避免核心类被篡改，也能避免类重复加载。

### 类加载过程是什么？

#### 标准回答

类加载过程包括加载、验证、准备、解析和初始化。加载是读取 class 字节码并生成 Class 对象；验证保证字节码安全；准备为静态变量分配内存并设置零值；解析把符号引用转成直接引用；初始化执行静态变量赋值和静态代码块。

## 对象回收与引用

### 如何判断对象是否可回收？

#### 标准回答

主流 JVM 使用可达性分析判断对象是否可回收。从 GC Roots 出发，如果对象无法通过引用链到达，就认为对象不可达，可以被回收。引用计数法简单但无法解决循环引用问题，因此 HotSpot 不依赖引用计数作为主要判断方式。

### Java 引用类型有哪些？

#### 标准回答

Java 引用类型包括强引用、软引用、弱引用和虚引用。强引用只要存在就不会被回收；软引用适合缓存，内存不足时可能被回收；弱引用只要发生 GC 就可能被回收；虚引用不能获取对象，主要用于跟踪对象回收和管理堆外资源。

### 对象什么时候进入老年代？

#### 标准回答

对象通常先分配在 Eden 区，经历 Minor GC 后仍然存活会进入 Survivor 区，多次存活达到年龄阈值后晋升老年代。大对象也可能直接进入老年代。如果 Survivor 空间不足，部分对象也可能提前晋升。

## 垃圾回收算法与收集器

### JVM 垃圾回收算法有哪些？

#### 标准回答

常见算法包括标记-清除、标记-整理、复制算法和分代收集。标记-清除容易产生碎片；标记-整理会移动对象减少碎片；复制算法适合存活对象少的新生代；分代收集根据对象生命周期把堆划分为新生代和老年代，采用不同回收策略。

#### 常见追问

- 新生代为什么适合复制算法？
- 老年代为什么不适合单纯复制算法？
- 什么是内存碎片？

### 常见垃圾回收器有哪些？

#### 标准回答

常见收集器包括 Serial、Serial Old、ParNew、Parallel Scavenge、Parallel Old、CMS 和 G1。Serial 偏单线程；Parallel 偏吞吐量；CMS 偏低停顿但会产生碎片且已废弃；G1 以 Region 为单位回收，兼顾吞吐和可预测停顿。

### CMS 垃圾回收器有什么特点？

#### 标准回答

CMS 以获取最短回收停顿为目标，主要阶段包括初始标记、并发标记、重新标记和并发清除。它使用标记-清除算法，停顿较短，但会产生内存碎片，也可能出现 Concurrent Mode Failure。CMS 在新版本 JDK 中已经被废弃。

#### 常见追问

### G1 和 CMS 有什么区别？

#### 标准回答

CMS 以降低老年代停顿为目标，但会产生内存碎片，且已经废弃。G1 将堆划分为多个 Region，可以按收益优先回收，并支持可预测停顿目标，是现代服务端应用的通用选择。

### 分代垃圾回收怎么工作？

#### 标准回答

新生代通常分为 Eden、From Survivor 和 To Survivor，默认比例常见为 8:1:1。对象优先分配到 Eden，Minor GC 时存活对象复制到 Survivor，年龄加 1，达到阈值后晋升老年代。老年代对象生命周期更长，空间不足时可能触发 Major GC 或 Full GC。

## JVM 调优与排查

### Full GC 频繁怎么排查？

#### 标准回答

先确认 GC 日志和监控指标，判断是老年代压力、内存泄漏、大对象分配还是元空间问题。然后结合 `jstat`、堆转储、MAT / JProfiler 分析对象占用和引用链。

### 常见 JVM 参数有哪些？

#### 标准回答

常见参数包括 `-Xms` 初始堆大小、`-Xmx` 最大堆大小、`-Xss` 线程栈大小、`-XX:NewRatio` 新生代与老年代比例、`-XX:SurvivorRatio` Eden 与 Survivor 比例、`-XX:+UseG1GC` 指定 G1、`-XX:+PrintGCDetails` 或新版 `-Xlog:gc*` 打印 GC 日志。
