# DevOps / 部署面试题

## Docker 解决什么问题？

### 标准回答

Docker 将应用和运行环境打包成镜像，解决环境不一致、部署复杂和迁移困难的问题。它不是虚拟机，而是基于容器隔离技术运行进程。

### 关联文档

- [Docker 容器化](/deploy/docker)

## Kubernetes 的核心能力是什么？

### 标准回答

Kubernetes 提供容器编排能力，包括服务部署、滚动发布、扩缩容、服务发现、配置管理和故障自愈。它适合服务数量较多、部署复杂度较高的场景。

### 关联文档

- [Kubernetes 编排](/deploy/kubernetes)

## CI/CD 流程通常包括什么？

### 标准回答

CI/CD 通常包括代码提交、依赖安装、单元测试、构建产物、构建镜像、推送镜像、部署环境和健康检查。目标是让交付流程自动化、可重复、可回滚。

### 关联文档

- [CI/CD](/deploy/cicd)

## 线上问题如何定位？

### 标准回答

先看告警和影响面，再查日志、指标、链路追踪和最近变更。定位思路是先恢复业务，再缩小范围，最后复盘根因和补监控。
