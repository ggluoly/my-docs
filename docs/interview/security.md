# 安全认证面试题

## Session 和 JWT 有什么区别？

### 标准回答

Session 状态保存在服务端，客户端只保存 Session ID；JWT 将用户身份和声明放在 Token 中，服务端可无状态校验。Session 更容易主动失效，JWT 更适合分布式和跨服务传递，但吊销和续期更复杂。

## Spring Security 核心流程是什么？

### 标准回答

请求进入过滤器链，经过认证过滤器、异常处理、权限判断等环节。认证成功后把用户信息放入安全上下文，后续鉴权根据用户身份和权限判断是否允许访问。

- [Spring Security](/security/spring-security)

## OAuth2 解决什么问题？

### 标准回答

OAuth2 是授权协议，用来让第三方应用在用户授权后访问受保护资源，而不需要直接暴露用户账号密码。常见模式包括授权码模式、客户端模式等。

### 关联文档

- [OAuth2 与 JWT](/security/oauth2-jwt)

## 接口鉴权通常怎么做？

### 标准回答

常见做法是登录后签发 Token，请求携带 Token，网关或服务端解析身份，再根据角色、权限、资源规则判断是否放行。微服务场景通常还要考虑网关统一鉴权和内部服务信任边界。
