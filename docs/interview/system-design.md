# 系统设计题

## 秒杀系统怎么设计？

### 回答思路

秒杀系统重点关注高并发、限流、库存一致性、异步下单和防刷。典型链路是网关限流、活动预热、Redis 扣库存、MQ 异步创建订单、数据库最终落库。

### 关键点

- 流量入口限流和验证码 / 风控。
- Redis 预热库存，Lua 保证原子扣减。
- MQ 削峰，订单异步创建。
- 消费端幂等和失败补偿。
- 数据库乐观锁兜底。

## 订单系统怎么设计？

### 回答思路

订单系统要关注状态流转、幂等、库存、支付回调、超时关闭和消息最终一致性。设计时先讲核心流程，再讲一致性和异常补偿。

### 关键点

- 订单状态机清晰。
- 创建订单、支付回调、取消订单都要幂等。
- 库存扣减要防超卖。
- 超时关闭可用延迟消息或定时任务。
- 关键事件通过 MQ 解耦。

## 权限系统怎么设计？

### 回答思路

权限系统通常围绕用户、角色、权限、资源构建，常见模型是 RBAC。微服务场景下可以在网关做统一认证，在服务内部做细粒度权限校验。

### 关键点

- 用户、角色、权限、资源建模。
- 登录认证和 Token 管理。
- 菜单权限、按钮权限、接口权限区分。
- 网关鉴权与服务内鉴权分工。

## 日志系统怎么设计？

### 回答思路

日志系统要解决采集、传输、存储、检索、告警和链路关联。业务日志要有 traceId，错误日志要保留上下文，日志平台负责集中查询和问题定位。

## 文件服务怎么设计？

### 回答思路

文件服务通常使用对象存储，业务库只保存文件元数据。上传要考虑文件大小、类型校验、权限、断点续传、预签名 URL 和 CDN 加速。
